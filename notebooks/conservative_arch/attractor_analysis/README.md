# `attractor_analysis/` -- Semantic-attractor extraction from $V_\theta$

This experiment tests one of the load-bearing claims of the *Semantic
Simulation* framework: that a trained scalar potential
$V_\theta(\xi, h)$ has isolated minima corresponding to coherent
semantic configurations (token distributions / topics).  Attention
transformers have no analogue of this prediction, so a positive result
would be a clean qualitative win for the simulation framing.

The conclusion of this study is more subtle than the original prediction:

1. **$V_\theta$ has no finite local minima.**  Pure Adam descent on
   $V_\theta(\xi, h)$ at fixed $\xi$ runs $h$ off to infinity:
   $\langle V\rangle$ reaches $-2500$ at step 300 (vs. $-260$ at the
   real trajectory) and never settles.  $V_\theta$ is unbounded below
   along multiple directions because it is trained only through its
   *gradient* (the force in the integrator); its absolute scale is
   never penalised.

2. **Anchored descent is unimodal and prompt-independent.**  Adding a
   data-manifold prior, $\frac{\lambda}{2}\lVert(h-h_c)/h_s\rVert^2$, makes the
   problem well-posed.  But for any $\lambda \in [10, 10^3]$, the
   landscape collapses to *one* dominant attractor whose decoded
   distribution is essentially $h_c$ itself -- the same five stopwords
   (`,`, `\n`, `the`, `a`, `-`) for every prompt.

3. **The damped dynamics at $L = L_\text{train}$ DOES exhibit
   prompt-dependent multi-basin structure.**  Running SPLM's own
   semi-implicit damped integrator from random $h$ seeds at fixed
   $\xi$, for exactly the training number of steps, gives
   $K^* \in \{2, \dots, 10\}$ silhouette-optimal clusters that decode
   to *real* tokens and depend on the prompt.

The "semantic attractors" of SPLM are therefore **time-bounded basins
of the damped flow**, not minima of the potential.  This is consistent
with the framework's mathematics (Sec. \"Inference of Semantic
Structures\"), but contradicts the narrower energetic reading some
readers might infer from the phrase \"semantic attractor\".

## Quantitative summary

Five short prompts are conditioned on (`narrative`, `mathematics`,
`scientific`, `dialogue`, `code`).  Each run uses 384 seeds = 128
Gaussian + 128 token-embedding + 128 perturbed real-$h_L$.

| Run | Mode | $K^\ast$ per prompt | Decoded character |
|-----|------|---------------------|-------------------|
| `euler_L8_dt1_dyn`     | dynamics, 8 steps  | 10, 2, 10, 10, 10 | content + punctuation: ` the`, ` I`, ` to`, `\n`, `:`, `,` |
| `verlet_L16_dt05_dyn`  | dynamics, 16 steps | 5, 3, 6, 2, 5     | mostly punctuation: `,`, `\n`, `:` |
| `verlet_L16_dt05_grad` | $V_\theta$ + anchor GD | 2, 2, 3, 2, 3 | unimodal; all clusters decode to ` ,\n the a -` |

The rich multi-attractor structure of the **Euler L=8** baseline -- including
basins that decode to ` the`, ` I`, ` to`, ` and`, the dominant
content-word continuations of Shakespeare -- is the strongest
qualitative evidence we have so far that SPLM does learn a
prompt-conditional attractor landscape, in the dynamical sense.

The reduced attractor count of the Verlet L=16 runs is consistent with
their slight perplexity regression: the more accurate but heavier-damped
integrator concentrates more probability mass in the global
\"punctuation manifold\" of Tiny Shakespeare.

During-training evolution (new, generated by
`train_with_snapshots.py` + `render_training_evolution.py`):

- `results/training_evolution_euler_shakespeare_<prompt>.png`
  -- a 7-panel grid showing the landscape at training steps
  0, 50, 200, 500, 1000, 2000, 4000 for a freshly retrained SARF-mass
  Euler model.  At step 0 $V_\theta$ is flat (range $\sim 10^{-5}$);
  by step 200 a basin begins to carve out; by step 500 the
  silhouette-optimal K drops from 10 (noise clusters on a flat
  landscape) to 2 (real basins); at step 4000 the $V$ range has grown
  to $\sim 1500$ and a deep canyon with distinct endpoints is
  established.

## Reproducing

From this directory:

```bash
# 1. Euler baseline (L=8, dt=1.0)
python3 attractor_extraction.py \
    --ckpt ../sarf_mass_variant/results/splm_sarfmass_logfreq_shakespeare_ckpt_latest.pt \
    --tag euler_L8_dt1_dyn \
    --mode dynamical --n_sim_steps 8 --sim_dt 1.0 \
    --n_gauss 128 --n_tok 128 --n_real 128 --K_max 10

# 2. Verlet L=16 dt=0.5 (small-dt variant)
python3 attractor_extraction.py \
    --ckpt ../symplectic_variant/results/splm_sym_logfreq_shakespeare_L16_dt05_ckpt_latest.pt \
    --tag verlet_L16_dt05_dyn \
    --mode dynamical --n_sim_steps 16 --sim_dt 0.5 \
    --n_gauss 128 --n_tok 128 --n_real 128 --K_max 10

# 3. Diagnostic gradient mode on the same Verlet checkpoint
python3 attractor_extraction.py \
    --ckpt ../symplectic_variant/results/splm_sym_logfreq_shakespeare_L16_dt05_ckpt_latest.pt \
    --tag verlet_L16_dt05_grad \
    --mode gradient --steps 1500 --lambda_reg 100 \
    --n_gauss 128 --n_tok 128 --n_real 128 --K_max 8

# 4. Comparison figure (3 x 5 bar-chart grid)
python3 make_comparison_figure.py

# 5. 3D inference-time landscapes (per model, per prompt)
python3 landscape_3d.py \
    --ckpt ../sarf_mass_variant/results/splm_sarfmass_logfreq_shakespeare_ckpt_latest.pt \
    --tag euler_L8 --prompts all --save_anim          # GIFs for all prompts
python3 landscape_3d.py \
    --ckpt ../symplectic_variant/results/splm_sym_logfreq_shakespeare_L16_dt05_ckpt_latest.pt \
    --tag verlet_L16_dt05 --prompts all

# 6. Euler-vs-Verlet side-by-side 3D comparison figure
python3 compare_landscapes_3d.py --prompts all

# 7. During-training evolution (retrain with snapshots; ~20 min on M2 MPS)
python3 train_with_snapshots.py --tag euler_shakespeare \
    --snapshot_steps 0,50,200,500,1000,2000,4000
python3 render_training_evolution.py \
    --snap_tag euler_shakespeare --prompt narrative
```

Outputs land in `results/`:
- `attractors_<tag>_<prompt>.png` -- per-prompt PCA + decoded centroid plot
- `attractors_<tag>_results.json` -- machine-readable per-prompt clustering
- `attractors_<tag>_summary.md`   -- human-readable markdown table
- `attractors_comparison.png`     -- 3 x 5 side-by-side comparison

## Key arguments

- `--mode {gradient, dynamical}`
  - `gradient`: Adam on $V_\theta(\xi, h) + \frac{\lambda}{2}\lVert(h-h_c)/h_s\rVert^2$.
    Diagnostic only; $V_\theta$ alone has no finite minima so the anchor
    is required.
  - `dynamical`: semi-implicit Euler on $\ddot{h} = -\nabla V_\theta(\xi, h)/m - \gamma \dot{h}$
    from random $h$ seeds.  Use `--n_sim_steps L_train` to reproduce
    the basins the model actually uses.
- `--lambda_reg`: anchor strength for `gradient` mode.  100 is a sane
  default; 10 gives weak convergence, 1000 trivialises the problem.
- `--n_sim_steps`: integration steps for `dynamical` mode.  Setting
  this above the training $L$ leads to runaway because $V_\theta$ is
  unbounded below.
- `--K_min, --K_max`: range of K for silhouette-sweep K-means.

## Files

- `attractor_extraction.py` -- main extraction script (both modes)
- `make_comparison_figure.py` -- builds the 3 x 5 bar-chart comparison figure
- `landscape_3d.py` -- 3D $V_\theta$ surface + trajectories for one
  checkpoint (per-prompt PNGs and optional rotating GIFs)
- `compare_landscapes_3d.py` -- Euler-vs-Verlet side-by-side 3D
  comparison on one figure per prompt
- `train_with_snapshots.py` -- retrain SARF-mass SPLM while saving
  intermediate checkpoints at log-spaced training steps
- `render_training_evolution.py` -- consume the snapshots and render
  the grid of $V_\theta$ landscapes at each training step
- `results/`    -- run outputs (PNG/GIF/JSON/MD)
- `results/snapshots/<tag>/` -- intermediate training checkpoints
- `README.md`   -- this file
